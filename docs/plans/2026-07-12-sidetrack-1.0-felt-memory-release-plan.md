## Thesis

Today Sidetrack has built the exact product the whole AI-browser market retreated to in H1 2026 (sidebar + cross-tab memory + org-aware context packs, all on-device) — but the 63k LOC of intelligence is invisible, unproven, and undistributed: the curated-memory moat literally never reaches the packet the user hands their AI (the real extension contextPack.ts has zero recall/neighbor references), the "nothing leaves the device" claim is asserted but not demonstrable, the six freeze-lift counters can't even be measured (all read "pending"), and the MCP is a 31-tool churn magnet nobody would keep installed (private:true, 0.0.0, bun-only, docs still describe the dead bac.* namespace). After this release, a stranger opens the side panel and immediately feels the memory: a durable "where was I?" resume rollup on every open, déjà-vu chips that explain themselves in plain language, context packs that inject their own curated cross-tab research (the moat, made physical), a one-click zero-outbound network-trace proof, and a one-command MCP with a single "search_my_research" tool their coding agent recalls decisions and rejected patterns from. Crucially, the release does not ship on demo vibes: it stands up the private, real-vault eval spine that PROVES the ranker beats a grep-over-vault baseline by ≥10 points (the field's non-negotiable bar) so the post-freeze serving improvements are earned by dogfood signal, not asserted. It is sequenced strictly around the two immovable gates — 16/16 §13 and the 30-day §15 window — so every freeze-safe surface, distribution, and eval build runs DURING the incompressible soak, and only the earned serving-math depth lands after the freeze lifts.

## Market research digest (mid-2026, live-web, per-finding citations)

### Agentic / in-browser AI copilots

Agentic / in-browser AI copilots, mid-2026. The category consolidated hard in H1 2026: OpenAI killed the standalone Atlas browser after ~9 months ("the browser is a feature, not the destination") and pushed its browser-agent into the ChatGPT desktop app + a Chrome extension; The Browser Company (Arc/Dia) was absorbed by Atlassian ($610M) and pivoted Dia to enterprise while Arc went to maintenance mode; Microsoft retired Edge "Copilot Mode" over low usage and folded it into mainline Edge; Google shipped Gemini-in-Chrome with Auto Browse + "Skills." Only Perplexity Comet is pushing a standalone AI browser at scale (went free on all platforms; ~47.6% of measured agentic traffic in June 2026). Cross-cutting reality: agentic "do things for me" is mostly a demo (checkout/payment ~2.3% of agent activity vs ~79% research/discovery); prompt injection is unsolved (Gartner told enterprises to block AI browsers; law firms banning them); and privacy of browsing memory is the biggest recurring user complaint. The winning retained patterns are the humble ones — sidebar chat with current-page context, page summarize/ask, and cross-session/cross-tab memory — not autonomous agents.

**Patterns that won:**
- Sidebar / side-panel chat with CURRENT-PAGE context — the single most-retained pattern across every player (kept even by the products that killed everything else: Atlas→ChatGPT extension, Edge post-Copilot-Mode, Brave Leo). This is exactly Sidetrack's side panel. Keep it central.
- Page summarize + ask-about-this-page — the reliable, daily-driver capability. Boring, unglamorous, universally retained.
- Cross-session / cross-tab MEMORY ('remember what I researched Monday, resume Thursday without re-briefing') — reviewers call this 'the quiet revolution of 2026.' It is Sidetrack's home turf (event-sourced vault + recall), and the privacy-clean version of it is unoccupied.
- Research / discovery workflows — ~79% of actual agent usage. Summarize technical docs, pull citations across open tabs, comparison research. This is the retained job-to-be-done, NOT transactions.
- Reusable saved workflows/prompts ('Skills' in both Chrome and Dia) — validates the context-pack idea; users want to save and re-invoke context, not re-type it.
- Extension / app form factor OVER standalone browser — the market's collective retreat (Atlas, Edge, effectively TBC) toward 'AI as a layer on the browser you already use' is Sidetrack's native shape.
- BYO / local-model option (Brave Leo) — privacy-conscious users actively want to point the assistant at their own local LLM; worth being first-class.
- Cross-device continuity feel — Comet reviewers valued sessions following them between devices (a UX expectation, even if Sidetrack's N=1 local design handles it differently).

**Patterns to avoid:**
- Building/forking a whole browser. Arc died, Atlas died, Edge Copilot Mode died — maintaining a Chromium fork is unsustainable and the AI value doesn't require it. Sidetrack correctly is NOT a browser.
- Autonomous agentic transactions (agent buys/books/checks-out for you). ~2.3% of real usage, ~65-70% task success at best ('30% comedy'), low user trust (54% prefer manual), legally contested (Amazon injunction), and the #1 prompt-injection exfil vector. Gimmick — do not build.
- Shipping browsing history / URLs / page content to a server, especially with no opt-out. Comet's autosuggest URL leak + ad-profiling + April 2026 lawsuit is the category's cautionary tale. Sidetrack's on-device design is the direct counter — never regress it.
- Hijacking the address bar / omnibox into a chat box. A top, recurring complaint against Edge Copilot Mode ('the address bar doesn't behave like an address bar'). Keep AI in the side panel.
- Server-side / silent feature re-enablement of capture. Edge got burned by toggles that re-enabled themselves. A privacy product must never surprise-enable data collection (Sidetrack's no-server design structurally prevents this — keep it that way).
- Omnipresent / intrusive AI UI. Edge Copilot Mode's low usage was partly 'it's everywhere and I didn't ask.' Make Sidetrack's presence calm and opt-in, not shoved into every surface.
- Ad-supported memory personalization. Comet builds ad-targeting profiles from memory; this is precisely the trust-killer to avoid — Sidetrack's N=1 local model has no ad incentive, which is a feature to state explicitly.
- Over-indexing on 'agent success rate' demos. The reliability gap ('impressive when it works, frustrating when it doesn't', 40% wrong-clicks on SPAs) burned Atlas's reputation. Under-promise on autonomy; over-deliver on recall/context.

**Implications for Sidetrack:**
- PRIVACY IS THE MARKET WEDGE, NOT A FEATURE. In mid-2026 the #1 user complaint about every cloud AI browser is browsing-memory privacy (Comet leaks all URLs with no opt-out and got sued; Gartner says block AI browsers; law firms banning). Sidetrack's 'nothing leaves the device, sole local writer, no telemetry' should be the LEAD claim. Concretely: build the Comet-style network-trace teardown ON Sidetrack showing zero outbound page/URL traffic, and put it in the README/demo. This is freeze-safe (surfacing/eval) and the highest-leverage marketing artifact available.
- THE FORM FACTOR IS VINDICATED — STOP SECOND-GUESSING IT. OpenAI ('browser is a feature not the destination'), Microsoft (killed Copilot Mode), and TBC (Arc→maintenance) all concluded the same thing Sidetrack is built on: AI belongs as a layer on the user's existing browser, not a new browser. The Chrome-MV3-extension + local-companion + MCP architecture is exactly where the category converged. Do not entertain any 'should we be a browser' pivot.
- DOUBLE DOWN ON RESEARCH/RECALL/ORGANIZE, IGNORE AUTONOMOUS TRANSACTIONS. ~79% of real agent usage is research/discovery; ~2.3% is transactions. Sidetrack's recall + connections graph + deja-vu strip + workstream org IS the retained job. Do not spend agent-waves on 'agent buys things.' The queued eval infra (replay harness, scored connections precision, ship-gate significance) directly strengthens the part of the product that actually retains users — prioritize it over any agentic-action surface.
- CONTEXT PACKS ARE THE MOAT — AND CHROME/DIA JUST VALIDATED THE DEMAND (with shallow versions). Both shipped 'Skills' (save/reuse prompts). Sidetrack's org-aware context packs from manually-curated workstreams + cross-tab graph + learned reranker are strictly deeper. Frame the pitch as 'Skills, but built from your actual curated work across all tabs, handed to your coding agent.' The 63k-LOC built intelligence is the differentiator that Gemini/Dia structurally won't replicate. Get the §13 demo closing this so the depth is visible.
- MCP-TO-CODING-AGENTS (ICP2) IS A REAL, GOOGLE-BLESSED LANE — BUT DIFFERENTIATE FROM DEVTOOLS-MCP. Chrome DevTools MCP already gives Claude Code/Cursor a live-tab debugging lens. Sidetrack's MCP must NOT compete on 'drive the browser'; it should hand agents CURATED MEMORY/CONTEXT (what the user has organized and cares about) that no DevTools MCP has. Distribute via the MCP registry as memory prep does — that's the wedge into the solo-dev + agent workflow, and it's freeze-safe distribution work.
- SECURITY POSTURE IS A STORY, NOT JUST A DEFAULT. Prompt-injection exfil (Brave's Comet OTP demos) is the agentic category's unsolved wound and the reason enterprises ban these tools. Sidetrack's read-mostly, user-in-the-loop design (context handed to an agent the USER drives, no autonomous write-tools acting on hidden page instructions) structurally avoids that exfil class. Say so explicitly — it turns a design constraint into a trust advantage, especially for the law-firm/enterprise-adjacent buyer that just banned everything else.
- AVOID THE UX ANTI-PATTERNS THAT KILLED THE OTHERS. Keep AI in the side panel (never hijack the omnibox — Edge's cardinal sin); keep presence calm/opt-in (Copilot Mode's 'everywhere' drove low usage); never silently re-enable capture (no-server design already prevents this — protect it). These are all freeze-safe surfacing/UX guardrails.
- MONETIZATION IS EASIER FOR SIDETRACK THAN FOR ANY CLOUD PLAYER — DON'T COPY THEIR MODEL. Comet had to go FREE and can't run ads (subscription-only); Perplexity is paying 80% rev-share to publishers to stay alive; everyone else eats server cost. An N=1 on-device tool has ~zero marginal serving cost, so it can be a one-time/BYO-model/open-core product without the growth-lever desperation that forced Comet free and Perplexity into publisher deals. Consider a first-class BYO-local-model story (à la Brave Leo) to widen the privacy-purist appeal.
- WALL-CLOCK HONESTY. Copying the retained patterns (sidebar/summarize/memory) is largely DONE in Sidetrack; the gap to competitors is the §13 demo + 30-day dogfood window, not missing features. Agent-waves can produce the eval infra, the privacy-trace teardown, and MCP-registry distribution prep quickly, but the ship gate (16/16 demo) and the 30-day soak are irreducibly serial. Frame roadmap as: verify → surface the built intelligence (freeze-safe) → distribute via MCP → then post-freeze serving improvements — which matches the existing memory roadmap, now externally corroborated by where the whole market landed.

### Personal memory / total-recall products

Personal memory / total-recall products, mid-2026 (screen-recording, audio-wearable, and local-first PKM lineages)

**Patterns that won:**
- DAILY DIGEST is the single most reliably sticky retrieval UX across the arena — Limitless's stickiest feature (email journal of highlights + action items) and Khoj's stickiness engine (scheduled automations). A recall product that PUSHES a curated summary retains; one that only offers a search box to PULL becomes shelfware.
- ACTION-ORIENTED RESURFACING beats raw log search: the winning outputs everywhere are reminders / to-do extraction / 'the 3 things that mattered today' (Limitless, Bee, Khoj) — hand the user the signal, don't make them fish through the noise.
- NARROW / STRUCTURED CAPTURE SCOPE tolerates better than screen-everything: event-based/structured capture is smaller, maskable (input fields auto-hidden), searchable by property, and dodges the noise-vs-signal wall. Browser-scoped sits in this sweet spot — it sees logged-in tabs and real engagement without the sousveillance/consent baggage of ambient mic or full-screen video.
- SCOPE CONTROLS AS A TRUST PRIMITIVE: Khoj's /notes (answer only from my corpus) and Recall's per-app/site exclusions are loved. Legible, user-held scope switches convert privacy anxiety into trust.
- OPT-IN + SOURCE-AVAILABLE + LOCAL is the trust triad that let OSS players (Screenpipe, Khoj) survive where a closed screen-recorder (Recall) stayed 'tainted' despite a full re-architecture.
- DISTRIBUTION VIA AGENT/DEV ECOSYSTEM survived where end-user recall UIs stalled: Screenpipe endures as the agent-memory API + 100 integrations; the coding-agent surface (Cursor/Claude Code/Cline) is a live, growing channel.
- RELIABILITY TRANSPARENCY: 'when recording is broken, the user should know ASAP' is a top Screenpipe issue — silent failure destroys trust in a memory product faster than missing features.

**Patterns to avoid:**
- SCREEN-EVERYTHING / RECORD-EVERYTHING capture: consistently loses. Recall <10% adoption and 'forever tainted'; Rewind 'toaster'; Screenpipe 700% CPU + 5-11 GB/month + consent objections; a week of Rewind = 'way too much noise vs signal to be useful.' The commodity screenshot-OCR clones (OpenRecall et al.) all hit the same wall and none broke out.
- ON-DEVICE CV/OCR PIPELINES AS THE COMPUTE MODEL: the CPU/battery drain was the #1 cited abandonment reason for Rewind and the top HN complaint for Screenpipe. Directly relevant to Sidetrack's 100-144% CPU-runaway history — the fix regime is existential, not cosmetic.
- THE BUILT-IN PROACTIVE AI ASSISTANT as the headline feature: explicitly abandoned in the Limitless review for 'lacking broader context.' Users route to Claude/Perplexity instead. Don't try to be the smart chatbot; be the context PROVIDER to capable external LLMs.
- IMPRESSIVE-DEMO-BUT-UNRELIABLE features: Rewind's 'Ask Rewind' 'walked the line of the accuracy needed for reliable use' and never crossed it — a prototype that never became load-bearing. Validates a hard acceptance-gate before dogfood, not a demo-driven ship.
- SELF-HOSTING / INSTALL FRICTION as an adoption ceiling: Khoj's recurring pain is Docker/dependency/OS setup failures. Sidetrack's Bun-companion install is the analogous risk — a broken install is a silent churn source.
- OPAQUE DATA PROCESSING: Rewind users 'lacked visibility into how data was processed, making improvement feel impossible.' Trust in a memory product compounds only if provenance/processing is legible.
- RELYING ON LOCAL-FIRST ALONE FOR TRUST: necessary but not sufficient. Recall was re-architected to local/encrypted/enclave and still distrusted (TotalRecall Reloaded, March 2026). The story must be provable and legible, and one architecture failure poisons the brand for years.

**Implications for Sidetrack:**
- BUILD THE DIGEST/RESURFACING SURFACE — this is the highest-ROI, freeze-SAFE (surfacing) move the whole arena points to. A scheduled 'here's what mattered' rollup over the vault (today's threads, unfinished workstreams, deja-vu revisits, 'you looked at X 3 weeks ago') is what made Limitless and Khoj sticky. The deja-vu related-pages strip is a proactive-resurfacing primitive Sidetrack already has — lean into it and add a pushed cadence, not just a pull surface. Note: this touches surfacing/UI only, NOT serving math, so it clears the P1 freeze.
- LEAD WITH THE BROWSER-SCOPED VANTAGE AS THE PRIVACY + SIGNAL STORY, not as a limitation. Sidetrack captures structured, scoped, real-engagement data — the tolerated sweet spot between screen-everything (Recall/Screenpipe failure mode) and hand-typed notes (Reor's ceiling). Position 'we see your logged-in tabs and real engagement, and NOTHING leaves the device' as the differentiator that Recall's reputation and Reor's manual-entry both lack.
- DO NOT MARKET A BUILT-IN ASSISTANT; MARKET CONTEXT-OUT. The arena's clearest UX flop is the underpowered native assistant. Sidetrack's org-aware context packs + MCP-to-Claude-Code/Codex/Cursor is exactly the pattern that WON (Screenpipe's agent-memory API). Make MCP-registry distribution ICP2's front door.
- THE CPU-RUNAWAY REGIME IS AN EXISTENTIAL CONSTRAINT, NOT A SOLVED BUG. Rewind ('toaster') and Screenpipe (700% CPU) died/struggle on exactly this. Keep the background-embed worker lane, don't flip on-main-loop embedding, and treat any regression here as ship-blocking. The 13.6% content-vector coverage ceiling is worth living with until the worker lane exists — better under-indexed than pegged.
- RETRIEVAL RANKING (SIGNAL-OVER-NOISE) IS THE WHOLE GAME — and it's the one thing the P1 freeze protects. 'Too much noise vs signal to be useful' killed screen-recorders. Sidetrack's LightGBM LambdaRank reranker + impression/label loop + the just-landed recsys data spine (PR #242) are the correct investment IF you can prove they beat the graph baseline. Prioritize the queued eval infra (replay harness over logged impressions, scored connections precision, paired-significance ship gate) — that's how you demonstrate signal-over-noise objectively, which no competitor did.
- SHIP-GATE DISCIPLINE IS VALIDATED BY THE FIELD. Rewind's fatal pattern was 'impressive prototype, never reliable enough.' The section-13 16-step acceptance demo as a hard gate before the 30-day dogfood window is exactly the antidote — do not ship on demo vibes. In agent-wave terms the eval-infra and digest surface are each ~1-2 waves, but the 30-day soak is irreducible wall-clock; sequence accordingly.
- MAKE PRIVACY LEGIBLE, NOT JUST TRUE. Recall proves local+encrypted claims alone don't earn trust. Surface provenance (the evidence-tier provenance on vectors/edges you just built), a visible 'nothing left the device' audit, and user-held scope controls (à la Khoj /notes and Recall exclusions). Legibility is freeze-safe and directly de-risks the ICP1 trust sale.
- KEEP THE COMPANION INSTALL ONE-COMMAND. Khoj's growth ceiling is self-host friction; the Bun daemon is your equivalent risk. A broken install is silent churn — invest in install-ux as distribution, not polish.
- DON'T COMPETE ON CAPTURE MECHANISM. OpenRecall-class screenshot-OCR clones commoditized capture and none broke out. Sidetrack's defensibility is the structured event vault + connections graph + learned reranker + curated-workstream-to-context-pack loop — the intelligence, not the recording. Message accordingly.

### Agent-memory infrastructure and temporal knowledge graphs

**Patterns that won:**
- Filesystem/markdown + agent-driven grep beats vector-DB and graph memory on the standard benchmark (Letta 74.0% vs Mem0 graph 68.5% on LoCoMo). 'The model is the index.' Anthropic's own production memory tool is text-file CRUD, not a vector store — the frontier lab bet on files.

- Write-time reconciliation (Mem0's ADD/UPDATE/DELETE/NOOP) and write-time entity resolution/merge — consolidate BEFORE storage, not at query time. 'Better consolidation earlier beats aggressive deletion.'
- Offline/idle 'sleep-time' or 'dreaming' consolidation that reorganizes memory during downtime, non-destructively (write a NEW store, keep the raw) — shipped by Letta, Anthropic (Dreams), and OpenAI (Dreaming V3, which took time-sensitive accuracy from 9.4% to 75.1%).

**Patterns to avoid:**
- Benchmark self-promotion: vendor-run LoCoMo/LongMemEval numbers are systematically inflated and get cut ~in half by independent re-runs (Zep 84%->58.4%; the Mem0/Zep 'benchmark theatre'). Never lead with a self-scored headline; the field now discounts them on sight.
- Treating LoCoMo (or any single static recall benchmark) as a sufficient bar: it has a 6.4% ground-truth error rate, a ~93.6% impossible-score ceiling, doesn't test knowledge UPDATES, and fits in modern context windows so full-context baselines beat 'memory systems.'
- Building graph/vector complexity that a filesystem+grep baseline can match — if the specialized stack doesn't beat grep-over-vault by >=10pts on the real data, the complexity is unjustified (the exact trap Letta exposed).
- Rewrite-in-place consolidation (OpenAI's older approach; criticized): iterative generative loops that overwrite the source drift and lose information. Consolidate into a new store and preserve raw (Anthropic Dreams pattern).
- Using an LLM as the arbiter of temporal supersession / contradiction (only 67.2% accurate) when deterministic rules hit 94.8%.
- Remember-everything with no importance gate: bloats context cost, dilutes attention, and floods top-k with near-duplicates — the classic 'Day-2 production failure' when real users return across sessions/days/topics.
- Hard-deletion for performance (vs merging/ranking staleness away): reserve eviction for GDPR/PII only; performance should come from consolidation and ranking.
- High-friction self-hosting (Zep's own-your-Neo4j path) — kills adoption for the local/solo segment.

### On-device / local-first ML + data stack for a privacy-first browser AI companion

On-device / local-first ML + data stack for a privacy-first browser AI companion, state of mid-2026. Sidetrack today: Xenova/multilingual-e5-small (384-dim, q8 ONNX via transformers.js) embeddings; sqlite-vec dense lane; LightGBM LambdaRank + online head learned reranker; leiden-cpm topics; single-writer Bun daemon over a local markdown+JSONL vault; MV3 side-panel extension; stateless MCP for coding agents. Local-first, single-user (N=1), no server component by design; on-device CPU (100-144% runaway history) and 2.4-3.6GB memory floor constraints stand.

**Patterns that won:**
- Purpose-built small models beat scaled-down big ones: EmbeddingGemma (308M, top-MTEB-under-500M), answerai-colbert-small (33M beats bge-base), mxbai-edge-colbert-17M (beats 130M ColBERTv2), with ONNX/transformers.js builds day-one.
- On-device inference lives where the data/process already are: browser-native Gemini Nano (free, zero-host) beats daemon-hosted models when the task fits the small band; ship feature-detection + graceful degradation.
- For short-text embeddings WASM-int8 quietly beats WebGPU; ONNX-on-CPU is the pragmatic winner, WebGPU reserved for batch/large-model work.
- Classic GBDTs held the tabular/LTR crown through 2026 for small-data ranking; the only neural threat (TabPFN-class) is compute-heavy and not a clear small-N win. Feature/label quality and eval infra move the needle, not model-class swaps.
- Embedded in-process vector stores (sqlite-vec) won on operational simplicity for local-first single-user apps; scaling to HNSW/IVF-PQ is an additive index decision made later.
- CRDT space matured to pick-by-ecosystem-and-semantics: Yjs for collab, Automerge for versioned history, Loro for speed, all production-grade MIT.

**Patterns to avoid:**
- Do NOT swap LightGBM for a neural LTR model at N=1: data volume far below where neural/TabPFN pays off, costs CPU Sidetrack cannot spare, and violates the P1 serving-math freeze.
- Do NOT move short-text embedding inference to WebGPU for speed: for single short inputs WebGPU is slower than WASM/CPU-ONNX; only batch background embedding could benefit, and even there WASM-int8 is the safe default.
- Do NOT treat Gemini Nano as a hard dependency: half of users lack it, it needs 22GB free disk and is evicted under 10GB, origin-trial permission names churn, content-policy blocks fire. Always feature-detect and degrade.
- Do NOT rush an EmbeddingGemma or any embedding swap during the freeze: it shifts the cosine features feeding the ranker, re-embeds the whole 450k-event vault via MODEL_ID revision, and burns a full rebuild. Not cleanly freeze-safe.
- Do NOT adopt a real-time-collab CRDT (Yjs) for the vault: Sidetrack is single-writer/event-sourced; append-only event-log replication fits far better.
- Do NOT chase multi-vector ColBERT minis as a LightGBM replacement: they do text-relevance reranking, not multi-signal feature fusion, and blow up per-token vector storage against sqlite-vec single-vector rows.
- Do NOT host a 1-2B local LLM in the Bun daemon for labeling when Chrome Nano covers the same band: it adds 1-2GB to a 2.4-3.6GB floor and reintroduces CPU-runaway risk for no gain.

**Implications for Sidetrack:**
- FREEZE-SAFE, HIGH-LEVERAGE NOW: add a Chrome Built-in AI (Summarizer/Prompt API) enhancement lane for why-chips, topic-labels, and workstream summaries, feature-detected, browser-side, zero daemon CPU, graceful fallback to existing leiden-cpm/token labels. Surfacing/UI (freeze-safe), moves summarization off the CPU-constrained daemon, improves the section-13 demo. ~1 agent-wave behind a flag; the gate is real-device availability testing, not build time.
- POST-WINDOW (serving-math, not during freeze): evaluate EmbeddingGemma-300M q8-ONNX as an embedding refresh. Best-grounded candidate, same runtime, but Matryoshka-truncate to 384 dims for sqlite-vec layout, budget a full 450k-event re-embed, re-tune ranker cosine features. Run as a shadow lane first (repo already has a shadow-serving pattern), gate on the replay-harness eval. Expect it to trip the P1 freeze; post-16/16 + post-30-day item.
- KEEP LightGBM LambdaRank, verified right for N=1. Redirect ranker energy to the queued eval/data-spine work (replay harness over logged impressions, IPW served_position, pointwise residual over the graph baseline, edge re-validation/TTL, gate-floor right-sizing). Feature/label quality and falsifiable eval are the bottleneck, not model-class. Most are freeze-safe (eval/collection/infra).
- KEEP sqlite-vec for now: below the brute-force pain threshold at Sidetrack's vector count and composes with the bun-sqlite driver constraint. Note HNSW-on-sqlite-vec or LanceDB IVF-PQ as a documented future scaling lever if per-query cosine ever dominates CPU at larger vaults; not worth the churn pre-scale.
- The background content-vector backfill lane (off, 13.6% coverage, title-only embeds) is the highest-value real win under the ceiling constraints, but the fix is an off-main-thread WORKER LANE (scheduling), not WebGPU. WASM/CPU-ONNX batch embedding on a worker thread; solving page-access + content coverage unlocks better dense recall and makes late-interaction rerankers viable later. Infra/collection (freeze-safe), likely the single biggest quality-per-effort lever.

If a full CRDT is ever needed, Automerge versioned-history fits the event-sourced vault best; Loro if raw perf dominates. Do not adopt Yjs.

## Pillars

### Pillar 1 — Gate & Prove (make the freeze-lift observable and the ranker falsifiable)

Turn 'we think it's better' into 'we can prove it' and make the freeze-lift condition measurable in real time. This is the pillar everything else is sequenced against: without the six §15 emit sites the window is unmeasurable, and without the eval spine the post-lift serving list is unshippable.

- §15 falsifiability counters: wire emit sites for all six PRD §15 counters (trackedSessionsFraction over 30d, ≥3 lossless move_item reorg sequences with bac_id-invariant check, ≥5 research_packet/coding_agent_packet audit dispatch events, ≥1 chrome.sessions.restore success, ≥1 context_pack streamable-HTTP call, ≥7 consecutive drain-lag/outbox-green days) — count from the durably-drained companion log (not the extension outbox), via typed reads / drain-time materialization to avoid the /status full-scan timeout class. Surface all six live in the Health panel.
- Freeze-safe eval spine on top of PR #242: a served-feature replay harness that re-scores the ranker over logged point-in-time impressions (replays real history, never re-derives against a drifted graph), computing nDCG@10/MRR/recall@k AND explicit-reject false-positive-rate on the REAL vault; a scored connections precision harness over the LIVE graph (replacing the synthetic FocusEvalPack that runs test-only at 0.85/gamma-0.18 vs the 0.90 that serves); paired-bootstrap significance ship-gate math + minimum-effect threshold (report-only until freeze lifts — building the gate is freeze-safe, flipping it into promotion is post-lift); a perf-regression lane asserting per-drain time budgets at REAL vault scale that reproduces a known incident shape (resolve-flood) before its green is trusted.
- The non-negotiable floor: add a grep-over-vault AND full-context baseline to the harness — the 63k-LOC ranker/graph/vector stack must beat it by ≥10 points on the user's real vault (Letta's filesystem+grep 74.0% vs Mem0-graph 68.5% is the field's bar). N=1 is an asset: this is a private, adversarially-judged eval on the user's own logged impressions, the field's most-defensible kind.
- Release-engineering gate: fix the normalized 13-fail/8-file extension unit baseline (F16) so the aggregate verify gate can pass, de-flake the cross-package §13 BDD e2e (F14) into a reliable CI gate, and reconcile the ADR-0010-says-auth-mandatory vs audit-F03-says-no-auth-by-default contradiction (a §13/security-legibility blocker) before any trust artifact ships.

**Exit criteria:** All six §15 counters emit and render live in the Health panel BEFORE the window opens. The replay harness produces a real nDCG@10/MRR/reject-FP verdict on the actual vault with the grep + full-context baseline in place; the significance gate renders a verdict (report-only, not promoting). The perf-regression lane fails a synthetic resolve-flood. The verify baseline is green and the §13 e2e is a reliable (non-flaky) CI gate. Do NOT publish any self-scored benchmark number externally.

### Pillar 4 — The Side-Door (win ICP2: solo dev + coding agent via a tiny, fresh MCP)
