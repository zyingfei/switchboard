Browser-AI Companion
Design Spec · v1.0 · April 2026 · Anchor surfaces / MVP

The switchboard for
your AI tabs.

A design specification for a Chrome extension and Obsidian-anchored memory ledger that observes every AI thread you have open, remembers the last three weeks of your research, and exposes that memory as an MCP server to your coding agents — without burning tokens, without uploading data, without becoming yet another sidebar AI.

Primary ICP Multi-provider AI
power user with Obsidian.

Surfaces Side panel · on-page · vault · localhost MCP.
Hard constraints Don't burn tokens · open architecture · privacy-first.

1

Active workstream is the only saturated element. Pulse-glow on the chip, signal-orange tint. Everything else is paper-and-ink. Eye learns immediately what's "now."

2

Provider tags are color-coded, not branded. Forest-green for ChatGPT, terracotta for Claude, slate-blue for Gemini — distinct without using anyone's logos (TOS-clean).

3

Status dots are semantic, not decorative. Pulsing signal-orange = unread reply (the highest-value signal). Pulsing amber = needs your action. Solid green = you replied last. Gray = stale.

4

Drift line is a row affordance, not a separate alert. Inline amber band on the affected thread shows the version mismatch and offers a one-click "push delta" — the spec's S7/S76 made tactile.

5

Capture rows show provenance inline. Cite-IDs (e3a7) make every saved chat turn referenceable from a written document via S134.

6

Déjà-vu card uses signal-orange because it's a live signal. The italicised serif body distinguishes it from utilitarian text — this is the surface where BAC says something interesting.

11

Native Obsidian Bases rendering. BAC writes the .base file once; Obsidian renders the table, filters, sorting, grouping. We never build that UI ourselves — the §23.3 scope simplifier in execution.

12

Lifecycle states are visible without a dedicated UI. Status pills (Unread reply / Waiting on AI / Stale / Decided) emerge from event-log queries — explicit lifecycle UI is deferred to v1.5 per the consolidated spec.

13

The YAML mirror makes BAC lock-out-proof. If the user uninstalls BAC tomorrow, every artifact is still a Markdown file with structured properties. Future tools can read the same shape.

14

Provenance fields are first-class. bac-cid (content address), bac-prev-hash (Merkle chain), bac-signature (Ed25519) — auditability shipped from day one, per §24.6.

04 / Moat

MCP both ways.

BAC exposes the user's cross-provider research memory as a localhost MCP server consumed by Claude Code, Cursor, and Codex. This is what makes BAC architecturally novel — server-side MCP servers can't reach into the user's logged-in browser sessions; BAC is the only thing that can. Read tools ship in v1, write tools follow in v1.5 behind an explicit opt-in.

06 / Safety

Screen-share-safe mode.

When navigator.mediaDevices.getDisplayMedia reports active screen capture, BAC masks bucket names, thread titles, and provenance metadata automatically. Nothing the user configured comes off the screen unless they explicitly opt out. Auto-on; user-overridable.
